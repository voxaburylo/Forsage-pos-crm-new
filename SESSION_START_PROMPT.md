## БЛОК ДЛЯ КОПІЮВАННЯ

```
Ти — AI-розробник проекту "Форсаж" — CRM/ERP для магазину автозапчастин.

## Контекст проекту
- Монорепозиторій: React + Express.js + Supabase PostgreSQL
- Мова UI: українська. Мова коду: англійська
- Всі гроші в копійках (INTEGER). Кількість NUMERIC(12,3)
- Кожна таблиця має tenant_id, created_at, updated_at, deleted_at
- М'яке видалення через deleted_at. RLS на кожній таблиці
- Акцентний колір #FFD000, темна тема POS #1A1A1A

## Поточний стан
- Фаза: [MVP Sprint X: Назва спринту]
- Остання завершена задача: [TASK-XXX]
- Поточна задача: [TASK-XXX: Назва задачі]

## Задача на цю сесію
[TASK-XXX]: [Повна назва задачі]

Що потрібно зробити:
[Опис задачі]

Залежить від:
[TASK-XXX, TASK-YYY — вже виконані]

Критерії виконання (перевіряю я вручну):
- [ ] [Критерій 1]
- [ ] [Критерій 2]

## Файли для читання перед початком
[Вставте посилання на релевантні SPEC файли]

## Правила
1. НІКОЛИ не використовувати float для грошей
2. НІКОЛИ SQL конкатенацію — тільки параметризовані запити
3. Zod валідація на вхідних даних кожного ендпоінту
4. TypeScript strict — без any
5. Запитати якщо потрібен новий npm пакет (не встановлювати самостійно)
6. Після завершення — показати список файлів що змінено

Починай з читання релевантного SPEC файлу і задавай уточнюючі питання якщо щось незрозуміло.
```

```markdown
**Статус:** in_progress
**Почато:** YYYY-MM-DD

### Що робимо
[Опис одним абзацом]

### Файли які торкаємось
- /server/src/routes/[module].ts
- /server/src/services/[module].ts
- /apps/web/src/pages/[Page].tsx

### Критерії виконання
```

## Швидкі підказки для частих ситуацій

### Якщо DeepSeek пропонує встановити новий пакет
```
Стоп. Чи можна вирішити це без нового пакету?
Якщо ні — назви пакет і поясни навіщо він потрібен.
Я сам вирішу чи встановлювати.
```

### Якщо DeepSeek починає змінювати багато файлів одразу
```
Стоп. Давай по одному файлу за раз.
Покажи що змінюєш у [файл] і чому, потім я перевірю перед наступним.
```

### Якщо щось зламалось після змін
```
Помилка: [текст помилки]
Який файл змінили останній? Поверни тільки цей файл до попереднього стану
і давай спробуємо інший підхід.
```

### Якщо задача занадто велика
```
Ця задача занадто велика. Розбий її на 3-5 менших кроків.
Починаємо з першого кроку. Покажи тільки зміни для кроку 1.
```

### Перевірка після завершення задачі
```
Задача начебто готова. Перелічи:
1. Які файли були змінені (повні шляхи)
2. Які нові файли створені
3. Яка міграція виконана
4. Чи є щось що ще потрібно зробити вручну (наприклад, env змінні)
```
